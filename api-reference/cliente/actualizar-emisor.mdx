---
title: "Actualizar Emisor"
description: "Actualizar datos de un emisor registrado"
---

Actualiza los datos de un emisor previamente registrado. La estructura del body es idéntica a la de [Registrar Emisor](/api-reference/cliente/registrar-emisor).

## Request

```
POST https://{empresa}.mipos.co.cr/api/actualizar/emisor
```

### Campos

Todos los campos son los mismos que en [Registrar Emisor](/api-reference/cliente/registrar-emisor):

<ParamField body="identificacion" type="number" required>
  Número de identificación del emisor a actualizar.
</ParamField>

<ParamField body="tipoIdentificacion" type="number" required>
  Tipo de identificación: `1` Física, `2` Jurídica, `3` DIMEX, `4` NITE.
</ParamField>

<ParamField body="codigoActividad" type="number" required>
  Código de actividad económica.
</ParamField>

<ParamField body="sucursal" type="number" required>
  Número de sucursal.
</ParamField>

<ParamField body="nombreComercial" type="string" required>
  Nombre comercial o razón social.
</ParamField>

<ParamField body="telefono" type="string" required>
  Número de teléfono.
</ParamField>

<ParamField body="correoElectronico" type="string" required>
  Correo electrónico.
</ParamField>

<ParamField body="ubicacion" type="object" required>
  Dirección física. Ver [Registrar Emisor](/api-reference/cliente/registrar-emisor) para la estructura.
</ParamField>

<ParamField body="ambiente" type="number" required>
  Ambiente: `1` Producción, `2` Pruebas.
</ParamField>

<ParamField body="usuarioHacienda" type="string" required>
  Usuario de acceso a comprobantes electrónicos.
</ParamField>

<ParamField body="claveHacienda" type="string" required>
  Clave de acceso (base64).
</ParamField>

<ParamField body="pinCertificado" type="string" required>
  PIN del certificado digital.
</ParamField>

<ParamField body="certificadoHacienda" type="string" required>
  Certificado `.p12` en base64.
</ParamField>
