## Base URL

Cada cliente tiene su propia URL de API. Reemplaza `{empresa}` con tu identificador:

```
https://{empresa}.mipos.co.cr
```

<Note>
  Todos los endpoints requieren autenticación. Incluye el header `Authorization: Bearer <token>` obtenido desde el [Auth Server](/api-reference/login).
</Note>

## Headers requeridos

| Header | Valor | Requerido |
|--------|-------|-----------|
| `Content-Type` | `application/json` | Siempre |
| `Authorization` | `Bearer <token>` | Siempre |

## Tipos de documento

| Código | Tipo | Descripción |
|--------|------|-------------|
| `"1"` | Factura Electrónica | Documento tributario con receptor identificado |
| `"3"` | Nota de Crédito | Documento para anular o ajustar una factura |
| `"4"` | Tiquete Electrónico | Documento simplificado sin receptor obligatorio |
