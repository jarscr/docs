---
title: "Tiquete Electrónico"
description: "Emitir un tiquete electrónico sin receptor identificado"
---

Envía un tiquete electrónico (tipoDoc `"4"`) al Ministerio de Hacienda. Se usa para ventas al consumidor final sin necesidad de identificar al receptor.

<Note>
  El campo `receptor` debe enviarse como string vacío `""` para tiquetes sin receptor.
</Note>

## Request

```
POST https://{empresa}.mipos.co.cr/api/enviar/items
```

### Campos principales

<ParamField body="emisor" type="string" required>
  Número de identificación del emisor registrado.
</ParamField>

<ParamField body="numeroReferencia" type="integer" required>
  Número consecutivo de referencia del documento.
</ParamField>

<ParamField body="codigoActividad" type="string" required>
  Código de actividad económica del emisor.
</ParamField>

<ParamField body="tipoDoc" type="string" required>
  Tipo de documento. Para tiquete electrónico: `"4"`.
</ParamField>

<ParamField body="receptor" type="string" required>
  String vacío `""` para tiquetes sin receptor.
</ParamField>

<ParamField body="folioType" type="string" required>
  Tipo de folio. Siempre `"INVOICE"`.
</ParamField>

<ParamField body="fechaEmision" type="string" required>
  Fecha y hora de emisión en formato ISO 8601.
</ParamField>

<ParamField body="detalleServicio" type="array" required>
  Líneas de detalle del documento. Ver [Factura Electrónica](/api-reference/cliente/enviar-factura) para la estructura completa.
</ParamField>

<ParamField body="otrosCargos" type="array">
  Cargos adicionales. Ver [Factura Electrónica](/api-reference/cliente/enviar-factura) para la estructura.
</ParamField>

<ParamField body="resumenFactura" type="object" required>
  Resumen con moneda y totales. Ver [Factura Electrónica](/api-reference/cliente/enviar-factura) para la estructura.
</ParamField>
